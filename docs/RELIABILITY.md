# 可靠性要求

本文档描述嵌入式固件的可靠性设计与运行期约束。

---

## 设计目标

- 上电后应能自检并通过 LED/串口报告状态。
- 关键外设初始化失败应进入 `Error_Handler()`，避免带病运行。
- 主循环不应阻塞超过 100 ms（除明确测试用途外）。

## 看门狗与异常处理

- 当前未启用独立看门狗（IWDG）。
- 后续若加入长时间运行任务，应启用 IWDG 并定期喂狗。
- `Error_Handler()` 当前为死循环，后续可扩展为复位或日志记录。

## 初始化顺序

1. 早期 GPIO（LED1）用于确认 MCU 运行。
2. HAL 初始化。
3. 系统时钟配置。
4. CubeMX 外设初始化（GPIO / FSMC / USART）。
5. 用户 BSP 初始化（LED / USART printf / LCD）。
6. 进入主循环。

## 时序约束

- LCD 复位：拉低 ≥1 ms，拉高 ≥120 ms。
- FSMC 读写时序以 LCD 数据手册为准，当前采用保守值。
- `HAL_Delay` 基于 SysTick，精度约 1 ms。

## 已知风险

- 当前主循环使用阻塞延时，不适合高实时性任务。
- 后续应引入时间片轮询或 RTOS（如 FreeRTOS）。
